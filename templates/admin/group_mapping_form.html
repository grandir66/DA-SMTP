Come funziona
Una rule dice "per ogni cliente sincronizzato dal gestionale, se il valore del campo X soddisfa la condizione Y, metti automaticamente il cliente nel gruppo Z".
L'auto-assignment scatta ad ogni sync (24h) e non tocca le membership manuali. Il record valutato e' il merge di campi canonici (codcli, contract_active, tipologia_servizio, ecc.) + campi raw originali del gestionale.

Cosa cercare nei dati del cliente

Nome esatto del campo da osservare. Puoi usare: campi canonici (codcli, ragione_sociale, contract_active, tipologia_servizio, contract_type, contract_expiry, timezone) oppure nomi originali del gestionale (es. priority_flag, cluster, aconto, is_active).
Per equals/contains: valore singolo (case-insensitive).
Per in_list: CSV (es. A,B,C).
Per regex: regex Python (es. (?i)^premium).
Per truthy/falsy/not_empty: ignorato.

Dove assegnare

🏛️ = gruppo built-in (concetti del prodotto). Crea nuovo gruppo.

Scope opzionale

Se selezionato, la rule si applica SOLO ai clienti che arrivano da questa specifica sorgente sync. Utile se hai gestionali diversi con campi diversi.
Numero piu' basso = valutata prima (rilevante solo tra rules che puntano allo stesso gruppo).
